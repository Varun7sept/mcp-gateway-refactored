# Step 3: Chat handler receives the request

The middleware passes. The HTTP router finds `POST /api/chat` is registered to `ChatHandler.HandleChat`. It calls the function.

---

## First: How is ChatHandler created?

In `routes.go:64`, the server creates a ChatHandler with all the pieces it needs:

```go
ch := &handlers.ChatHandler{
    Gateway:   s.gateway,      // MCP gateway to call tools
    Logger:    s.logger,       // request logger
    Brain:     s.brain,        // AI Manager (the brain!)
    Auth:      s.auth,         // JWT auth
    ChatStore: s.chatStore,    // MongoDB chat storage
    MemHistory: s.memHistory,  // in-memory fallback
}
```

And `Brain` is set in `main.go:99-102`:

```go
groqKey := os.Getenv("GROQ_API_KEY")
if groqKey != "" {
    brain = ai.New(groqKey)      // only created if API key exists
}
```

**So `Brain` is nil if GROQ_API_KEY is not set.** The check `if h.Brain == nil` just checks: "Was the API key provided at startup?"

---

## The 3 checks inside HandleChat

### Check 1: Is AI configured? (line 29)

```go
if h.Brain == nil {
    writeJSON(w, 503, {"error": "AI not configured (set GROQ_API_KEY)"})
    return
}
```

`h.Brain` is a pointer to `ai.Manager`. If the `GROQ_API_KEY` environment variable was NOT set when the server started, this pointer is `nil` (empty). The request stops — 503 Service Unavailable.

### Check 2: Is the message valid? (line 38)

```go
var req struct {
    Message   string   `json:"message"`
    SessionID string   `json:"session_id"`
}
if err := json.NewDecoder(r.Body).Decode(&req); err != nil || req.Message == "" || req.SessionID == "" {
    writeJSON(w, 400, {"error": "message and session_id required"})
    return
}
```

It tries to read the JSON body into a struct. If:
- The JSON is malformed (error) → 400
- `message` is empty → 400
- `session_id` is empty → 400

### Check 3: Is message too long? (line 42)

```go
if len(req.Message) > 10000 {
    writeJSON(w, 400, {"error": "message too long"})
    return
}
```

Limits to 10,000 characters. Prevents spam/abuse.

## About sessionID — It is NOT from JWT

The `session_id` is generated by the **frontend JavaScript**, not from JWT.

In `chatui.html:70`:

```javascript
let sessionId = 'local-' + Date.now();
```

It's just a timestamp-based random ID like `local-1745678901234`. This groups messages into conversations so the AI can see previous messages in the same chat.

**JWT identifies WHO you are. session_id identifies WHICH CONVERSATION this belongs to.**
