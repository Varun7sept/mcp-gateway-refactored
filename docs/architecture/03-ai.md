# AI Layer — internal/ai

## Why it exists

The AI layer translates natural language requests into tool calls,
manages LLM interactions, stores conversation memory, and synthesises
final answers — all through a single `Manager` entry point.

## Who calls it

- `internal/web/handlers/chat.go` (chat handler)

## Responsibilities

| Component | File | Role |
|-----------|------|------|
| Manager | manager.go | Orchestration: `RunAgent`, `DecideAction`, `SynthesizeAnswer` |
| Planner | planner.go | Lightweight keyword-based task breakdown |
| LLM | llm.go | HTTP calls to Groq API, model fallback chain |
| Tool Executor | tool_executor.go | Tool definitions sent to LLM |
| Memory | memory.go | In-memory store with keyword scoring |
| Prompt Builder | prompt.go | System prompt, synthesis prompt, retry prompt |
| Helpers | helpers.go | `stripThink`, `truncate` utilities |

## Public API

```go
func New(apiKey string) *Manager
func (m *Manager) Memory() *MemoryStore
func (m *Manager) DecideAction(userMessage string, history []map[string]string) (*ToolCallResult, error)
func (m *Manager) SynthesizeAnswer(userMessage, toolName, toolCallID, toolResult string) (string, error)
func (m *Manager) RunAgent(userMessage string, history []map[string]string, callTool func(string, map[string]any) (string, error)) (string, []common.ToolStep, error)
```

## Agent Loop

1. Planner generates a suggested execution plan based on keywords
2. Plan is injected into the system prompt as guidance
3. Loop (max 5 iterations):
   - Call LLM with current message history + tool definitions
   - If LLM returns a tool call → execute via callTool callback
   - If LLM returns text → done
4. On exhaustion, force a final summarisation call

## Model Fallback

Models are tried in order: `llama-3.3-70b-versatile` →
`qwen/qwen3-32b` → `qwen/qwen3.6-27b`.  Configurable via
`GROQ_MODELS` env var (comma-separated).
